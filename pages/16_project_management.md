# Task Identification

::content::

- **Task Identification**: The process of breaking down a project into smaller, manageable tasks. This involves identifying all the activities that need to be completed to achieve the project goals.
    - For us this is a process of working backwards from the SAT Criteria to identify every task we need to complete as part of the project.
    - Each task should be part of a larger milestone (e.g. SAT Criterion 1) and should be as specific as possible (e.g. "Create Use Case Diagram for Login System" rather than "Create Use Case Diagrams").
    - Tasks should be organised into the Problem Solving Methodology stages (Analysis, Design, Development, Evauluation) and activities to demonstrate your understanding of how the project holds together.

# Task Sequencing

::content::

- **Task Sequencing**: The process of determining the order in which tasks should be completed. 
    - Some tasks will have **dependencies** (e.g. you can't create a Gantt chart until you've identified the tasks) and these need to be sequenced correctly.
    - Some tasks will be **parallel** (e.g. you can work on the Use Case Diagram and the Data Flow Diagram at the same time) and these can be sequenced in any order.
    - Some tasks will be **milestones** (e.g. completing the Project Plan) and these should be clearly identified in the Gantt chart.
    - Milestones are 0 length tasks, they represent significant points in the project (e.g. completion of a SAT Criterion)

# Time Allocation

::content::

**Time allocation** is the process of estimating how long each task will take to complete. This is an important part of project management as it allows you to create a realistic schedule and ensure that you have enough time to complete all tasks before the deadline.

- Creating a Gantt chart allows you to visually represent the time allocation for each task and see how they fit together in the overall project timeline.
- Time allocation should be based on your understanding of the task and your own working style. It's important to be realistic and allow for some buffer time in case tasks take longer than expected.
- Time allocation can be adjusted as the project progresses and you gain a better understanding of how long tasks are taking.
- Realistic is the goal - not accurate. Nobody estimates time perfectly and you are not being assessed on perfection.

# Dependencies

::content::

**Dependencies** are the relationships between tasks that determine the order in which they must be completed. Understanding dependencies is crucial for effective project management as it helps to ensure that tasks are completed in the correct sequence and that resources are allocated efficiently.

- Think carefully about dependencies when sequencing tasks. If Task A must be completed before Task B can start, then Task A is a dependency for Task B.
- Eg. Before I leave the house, I need to get dressed. Getting dressed is a dependency for leaving the house.
- In a Gantt chart, dependencies are represented by arrows connecting tasks. This visual representation helps to clarify the relationships between tasks and can assist in identifying potential bottlenecks or scheduling conflicts.

# Critical Path

::content::

When the project plan is complete, we can identify the **critical path**. This is the sequence of tasks that determines the minimum time required to complete the project. If any task on the critical path is delayed, the entire project will be delayed. 

- Identifying the critical path allows project managers to focus on the most important tasks and ensure that they are completed on time.
- In a Gantt chart, the critical path is often highlighted to make it easier to identify and monitor throughout the project.
